# MatriChalvos · Sistema de Gestión Matrimonial

Versión preparada para publicar en GitHub Pages y visualizar desde iPhone.

## Archivos

- `index.html`: versión lista para GitHub Pages. Es el archivo principal que debe quedar en la raíz del repositorio.
- `MatriChalvos_v4_original_sin_modificar.html`: respaldo exacto del archivo original, sin modificar.
- `.nojekyll`: evita que GitHub Pages procese el sitio con Jekyll.

## Uso

1. Crear un repositorio en GitHub, por ejemplo `matrichalvos`.
2. Subir estos archivos a la raíz del repositorio.
3. Ir a **Settings > Pages**.
4. En **Build and deployment**, seleccionar **Deploy from a branch**.
5. Elegir branch `main` y carpeta `/root`.
6. Guardar.
7. Abrir la URL que entrega GitHub Pages desde Safari en iPhone.
8. En Safari, tocar **Compartir > Agregar a pantalla de inicio** para usarlo como web app.

## Importante sobre privacidad

Este archivo contiene datos personales reales del matrimonio e invitados. Si publicas el repositorio como público, cualquier persona con el enlace podría acceder. Para uso privado, usa un repositorio privado si tu plan permite GitHub Pages privado o considera Netlify/Vercel con control de acceso.

## Persistencia de datos

La app guarda cambios en `localStorage` del navegador. Eso significa que los cambios hechos desde el iPhone quedan guardados en ese iPhone/Safari. Para mover datos entre dispositivos, usa **Backup & restaurar** dentro de la app.
